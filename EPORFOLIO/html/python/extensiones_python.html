EXTENSIONES_VSC_PYTHON_RA1

1. A mini-summary of what to do in the challenge:

For this challenge, I selected and installed a set of essential extensions to work with Python in Visual Studio Code. I chose all the recommended ones because, out of the four, three are specific to Python (Python, Pylance, Python Environments) and the fourth is Live Server, which is necessary to view the web output of the work done.

2. Two complex concepts that you have learned:

  • Debugging in Python: With the main "Python" extension, I learned the importance of having an integrated debugger to inspect code execution and find errors efficiently.
  • IntelliSense with Pylance: I understood how the Pylance extension provides suggestions and code completions while programming, which speeds up development and helps avoid syntax errors.

3. How have you felt:

I felt that these tools were very necessary to set up a proper development environment. Having suggestions and environment management makes coding in Python much smoother.

4. What would you improve about your solutions:

I would improve the documentation by exploring more advanced features of the "Python Environments" extension, such as managing virtual environments (venv or conda) directly from the VS Code interface.

5. Technical words in English for the glossary:

  • Debugger: A computer program used to test and debug other programs (the "target" program).
  • Suggestions (IntelliSense): A code-completion aid that includes a number of features: List Members, Parameter Info, Quick Info, and Complete Word.
  • Environment Management: The process of managing different Python versions and dependencies (libraries) separate from the system-wide Python installation.
  • Live Server: A development tool that launches a local development server with live reload feature for static and dynamic pages.

6. What grade would you give the practice statement and why (0 to 10), and what would you improve:

I would give this practice a 9 out of 10. It is essential for setting up the workspace, though seeing practical examples of debugging would be a good addition.
