COMAN2_LINUX_RA1

1. A mini-summary of what to do in the challenge:

This practice focused on advanced user and file management in Linux. The first phase involved creating users and groups, assigning specific permissions, and working with special permission bits like SGID (to inherit group ownership) and Sticky Bit (to protect files in shared directories). We also configured default file permissions using Umask. The second phase focused on information retrieval and file searching using powerful command-line tools like grep, find, and locate.

2. Two complex concepts that you have learned:

  • Special Permissions (SGID & Sticky Bit): I learned that standard Read/Write/Execute permissions are not enough for collaboration. SGID ensures that new files created in a directory belong to the directory's group (enabling team access), while the Sticky Bit prevents users from deleting each other's files in shared folders like /tmp.
  • Umask Configuration: I understood how the umask value acts as a filter for default permissions. It subtracts permissions from the system default (666 for files, 777 for directories), allowing administrators to enforce a baseline security policy for new files automatically.

3. How have you felt:

I felt very competent managing the system via the terminal. Understanding how permissions affect security and collaboration gave me a clearer view of Linux system administration. The search commands section was also very useful for navigating the file system efficiently.

4. What would you improve about your solutions:

While I executed the commands manually, I would improve the solution by creating Bash scripts to automate the user and directory creation process. This would reduce the chance of human error and make the setup reproducible. Additionally, I would explore using ACLs (Access Control Lists) for more granular permission settings beyond the standard Owner/Group/Others model.

5. Technical words in English for the glossary:

  • SGID (Set Group ID): A special permission bit that allows files created within a directory to inherit the group ownership of that directory.
  • Sticky Bit: A permission bit set on a directory that restricts file deletion; only the file's owner, the directory's owner, or root can delete the file.
  • Grep: A command-line utility for searching plain-text data sets for lines that match a regular expression.
  • Umask: A command that determines the settings of a mask that controls which file permissions are set for files and directories when they are created.
  • Wildcard: A character (like *) that can be used to substitute for any other character or characters in a string, useful in commands like find or locate.

6. What grade would you give the practice statement and why (0 to 10), and what would you improve:

I would give this practice a 9 out of 10. It covers essential system administration tasks. To make it perfect, I would add a section on troubleshooting permission errors, as that is a common real-world scenario.
