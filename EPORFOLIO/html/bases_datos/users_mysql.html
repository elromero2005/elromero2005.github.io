USERS_MYSQL_RA23

1. A mini-summary of what to do in the challenge:

The challenge involved managing users and privileges in a MySQL database. The main tasks included creating users (such as 'Bego', 'Mati', 'Mifli', and 'Crispula') with specific passwords and defining their allowed connection hosts (localhost or a specific domain like 'lasalleinstitucion.es'). Subsequently, we practiced granular permission assignment using GRANT statements to allow specific actions (SELECT, UPDATE, DELETE) on specific tables, as well as revoking these permissions with REVOKE and managing security by changing passwords and deleting users.

2. Two complex concepts that you have learned:

  • Privilege Management (GRANT/REVOKE): I learned how MySQL allows detailed control over security, enabling permissions to be granted globally or per table. I understood how to use GRANT to assign permissions (including the ability for a user to grant permissions to others with WITH GRANT OPTION) and REVOKE to remove them.
  • Authentication and Host: I understood the importance of the 'user'@'host' combination. The same username can have different permissions or passwords depending on whether they connect from localhost or a remote domain, which we observed when trying to connect with the user 'Crispula' from a non-permitted location.

3. How have you felt:

I felt competent while performing the practice. At first, the permission syntax seemed dense, but by running connection tests and seeing how actions were denied or allowed based on the configuration, I understood the underlying security logic.

4. What would you improve about your solutions:

I would improve the preliminary check of connection environments. In one of the exercises (the 'Crispula' case), I tried to access from localhost when the user was restricted to the 'lasalleinstitucion.es' domain, which generated an access error. I would also be more careful when modifying system tables directly (the user table), as I learned that changing the password directly without using the encryption function causes authentication errors.

5. Technical words in English for the glossary:

  • GRANT: SQL statement used to provide access or privileges on database objects to users.
  • REVOKE: SQL command used to remove access rights or privileges previously granted to a user.
  • FLUSH PRIVILEGES: Command that reloads the grant tables in the MySQL server memory; it is necessary when modifying permission tables directly (such as INSERT or UPDATE in the mysql.user table).
  • Localhost: Standard hostname referring to the loopback IP address (127.0.0.1), indicating that the connection is made from the same machine.
  • DCL (Data Control Language): A syntax similar to a computer programming language used to control access to data stored in a database (Authorization).

6. What grade would you give the practice statement and why (0 to 10), and what would you improve:

I would give the assignment a 9 out of 10. The practice covers the fundamentals of database security very well, although I would add an exercise on recovering forgotten passwords to complete the training.
