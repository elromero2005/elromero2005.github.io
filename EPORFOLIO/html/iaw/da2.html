DA2_RA15

1. A mini-summary of what to do in the challenge:

The challenge consisted of developing a dynamic web application using PHP to simulate a dice game. The project involves two main files: an initial HTML form (da01.php) where the user selects the number of dice (1, 2, or 3), the number of faces per die (4, 6, 8, 10, 12, 20), and sets the opponent's score. The second file (da02.php) processes this data via the POST method, generates random numbers for the dice using random_int(), calculates the total sum, and determines the winner by comparing the result against the opponent's score.

2. Two complex concepts that you have learned:

  • Server-Side Logic with PHP Forms: I learned how to collect user input using the global $_POST array and cast variables to integers ((int)$_POST[...]) to perform mathematical operations safely on the server side.
  • Randomness and Conditionals: I understood how to generate cryptographically secure random integers using the random_int() function and how to use nested if/elseif structures to handle different game scenarios based on the user's configuration.

3. How have you felt:

I felt satisfied seeing the logic work correctly and being able to create a functional interactive game. It was interesting to see how backend logic connects with the frontend interface.

4. What would you improve about your solutions:

The code in da02.php is quite repetitive, using many if/elseif blocks for every combination of dice and faces. I would improve this by using Loops (for/while) to generate the dice rolls dynamically. This would drastically reduce the number of lines of code and make the application scalable to any number of dice without writing new conditions manually.

5. Technical words in English for the glossary:

  • Superglobal: Built-in variables in PHP (like $_POST) that are always available in all scopes.
  • Casting: The process of converting a variable from one data type to another, for example, converting a string input to an integer.
  • Backend: The part of the website that runs on the server (PHP in this case) and is not visible to the user.
  • Integer: A whole number (not a fraction) that can be positive, negative, or zero.
  • Request Method: The method used to send data to the server, such as POST or GET.

6. What grade would you give the practice statement and why (0 to 10), and what would you improve:

I would give the assignment an 8 out of 10. It is a good introduction to PHP forms and logic, but it could include instructions to use loops to encourage cleaner code practices from the start.
